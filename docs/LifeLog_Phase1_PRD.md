## 1. Executive Summary

**LifeLog**는 발달장애인이 영유아기부터 노년까지 생애 전 주기의 기록을 하나의 플랫폼에 담고, 당사자 본인이 그 기록의 주체가 될 수 있도록 지원하는 B2G 복지 SaaS 플랫폼이다.

현재 국내 발달장애인 지원 체계는 기관 전환마다 기록이 단절되고, 보호자가 모든 의사결정을 독점하며, 기관별로 표준화되지 않은 서식을 사용하는 구조적 문제를 안고 있다. 이로 인해 동일한 평가가 반복되고, 당사자의 목소리는 지원 기록에서 배제된다.

Phase 1은 **생애주기별 맞춤 기록 템플릿 제공**, **다중 기록자 협업 권한 체계**, **만 18세 법적 기준 권한 자동 이양** 세 가지 핵심 기능을 6개월 내 MVP 형태로 구축한다. 파일럿 대상 기관 3곳과의 PoC를 거쳐 공식 론칭하며, Phase 2에서 AI 분석·기관 간 API·모바일 네이티브 앱으로 확장한다.

## 2. 문제 정의

### 2.1 핵심 문제

발달장애인은 출생부터 노년까지 병원·특수학교·복지관·직업재활센터·자립지원센터 등 수십 개의 기관을 거친다. 그러나 기관 간 기록 이전은 사실상 불가능하며, 매번 초기 평가를 처음부터 반복해야 한다. 보호자가 기억하지 못하거나 서류를 분실하면 해당 기록은 영구 소실된다.

더 근본적으로는, **당사자 본인이 자신의 기록에 접근하거나 수정할 권한이 없다.** 만 18세 이후 법적 성년이 되어도 결정권 이양을 위한 제도적·기술적 장치가 부재하다.

### 2.2 영향 받는 사용자

| 사용자 | 문제 빈도 | 현재 대안 |
|--------|---------|---------|
| 보호자 | 기관 전환 시마다 (평균 연 1~2회) | 종이 서류 직접 보관, 반복 제출 |
| 당사자 | 생애 전반 (의사결정 배제) | 없음 |
| 사회복지사 | 케이스 관리 매일 | 엑셀·개인 메모 |
| 특수교사 | IEP 수립 시마다 (연 1~2회) | 종이 IEP → 전산 재입력 |
| 활동지원사 | 서비스 제공 매일 | 종이 일지·기관 자체 양식 |

### 2.3 해결하지 않을 경우의 비용

- **당사자:** 자기결정권 박탈 지속, 지원서비스 중복·누락 심화
- **보호자:** 기관 전환마다 동일 서류 반복 제출 (평균 20~30종)
- **지원인력:** 비표준화 서식으로 인한 행정 시간 낭비 (주 평균 3~5시간 추정)
- **사회적:** 발달장애인 권리보장법 취지 미실현, B2G 수주 기회 손실

## 6. 사용자 스토리

### 6.1 계정 및 인증

**US-001** — 역할별 회원가입
> 보호자로서, 자녀를 위한 기록 플랫폼에 보호자로 가입하고 싶다. 그래야 자녀의 생애 기록을 통합 관리할 수 있다.

**수락 기준:**
- [ ] 가입 시 역할 선택(보호자/당사자/전문인력) 화면이 제공된다
- [ ] 이메일 주소와 비밀번호로 계정을 생성할 수 있다
- [ ] 이메일 인증 링크를 통해 계정이 활성화된다
- [ ] 이미 존재하는 이메일로 가입 시 명확한 오류 메시지가 표시된다
- [ ] 비밀번호는 최소 8자, 대·소문자·숫자·특수문자 조합 조건이 표시된다

---

**US-002** — 전문인력 기관 인증
> 사회복지사로서, 내 소속 기관을 인증하여 플랫폼에 가입하고 싶다. 그래야 신뢰할 수 있는 전문인력으로 보호자에게 초대받을 수 있다.

**수락 기준:**
- [ ] 전문인력 가입 시 역할 선택(사회복지사/특수교사/일반교사/활동지원사) 화면이 제공된다
- [ ] 기관코드 입력 후 실시간으로 기관명이 확인된다
- [ ] 기관코드가 유효하지 않으면 "유효하지 않은 기관코드입니다" 오류가 표시된다
- [ ] 자격증 번호는 선택 입력이며, 입력 시 프로필에 표시된다
- [ ] 가입 완료 후 관리자 승인 또는 기관코드 자동 승인 처리된다

---

**US-003** — PASS 간편인증 로그인
> 보호자로서, 매번 이메일·비밀번호를 입력하지 않고 PASS 앱으로 빠르게 로그인하고 싶다. 그래야 이동 중에도 빠르게 기록을 확인할 수 있다.

**수락 기준:**
- [ ] 로그인 화면에 "PASS 간편인증" 버튼이 표시된다
- [ ] PASS 인증 성공 시 자동으로 플랫폼에 로그인된다
- [ ] PASS 인증 실패 시 "인증에 실패했습니다. 다시 시도해주세요" 메시지가 표시된다
- [ ] PASS 미연동 계정은 이메일 로그인 후 PASS 연동을 설정할 수 있다

---

### 6.2 당사자 프로파일 및 생애주기

**US-010** — 당사자 프로파일 등록
> 보호자로서, 자녀의 기본 정보와 장애 정보를 플랫폼에 등록하고 싶다. 그래야 생애주기별 기록을 체계적으로 시작할 수 있다.

**수락 기준:**
- [ ] 이름, 생년월일, 성별을 필수 입력할 수 있다
- [ ] 생년월일 입력 후 현재 생애주기(영유아기/학령기/청소년기/청년기/중장년기)가 자동 계산되어 표시된다
- [ ] 장애유형, 장애등급, 장애인등록번호는 선택 입력이며, 입력 시 암호화 저장 안내가 표시된다
- [ ] 프로파일 등록 완료 시 만 18세까지의 D-day가 표시된다
- [ ] 한 보호자가 여러 당사자 프로파일을 등록·관리할 수 있다

---

**US-011** — 생애주기 자동 갱신
> 보호자로서, 자녀의 생일이 지나 새로운 생애주기로 전환될 때 자동으로 반영되기를 원한다. 그래야 새로운 단계에 맞는 기록 템플릿을 즉시 사용할 수 있다.

**수락 기준:**
- [ ] 생일 다음날 자동으로 생애주기가 갱신된다 (스케줄러 기반)
- [ ] 생애주기 전환 시 보호자에게 앱 내 알림 및 이메일이 발송된다
- [ ] 새 생애주기에 해당하는 템플릿이 기록 작성 화면에 자동으로 표시된다
- [ ] 이전 생애주기의 기록은 타임라인에 그대로 유지된다

---

### 6.3 기록 템플릿 및 작성

**US-020** — 생애주기별 템플릿 선택
> 특수교사로서, 내 역할과 현재 학생의 생애주기에 맞는 기록 템플릿을 빠르게 찾고 싶다. 그래야 불필요한 항목 없이 필요한 내용만 입력할 수 있다.

**수락 기준:**
- [ ] 기록 작성 시 카테고리(건강·의료/교육·발달/일상생활/서비스 연계/자기표현) 선택 화면이 먼저 표시된다
- [ ] 카테고리 선택 후 현재 생애주기와 내 역할에 맞는 템플릿 목록만 표시된다
- [ ] 특수교사는 '교육·발달' 카테고리에서 IEP 템플릿을 선택할 수 있다
- [ ] 내 역할에서 작성 권한이 없는 템플릿은 목록에 표시되지 않는다
- [ ] 각 템플릿에 간략한 설명과 예상 입력 시간이 표시된다

---

**US-021** — 기록 작성 및 저장
> 사회복지사로서, 담당 케이스의 서비스 연계 내용을 표준화된 양식에 입력하고 저장하고 싶다. 그래야 이후 기관 간 정보 전달 시 동일한 내용을 재입력하지 않아도 된다.

**수락 기준:**
- [ ] 폼 입력 중 자동 임시저장이 5분마다 실행된다 (수동 저장 버튼도 제공)
- [ ] 필수 항목 미입력 시 저장 시도 때 해당 항목에 오류 표시가 된다
- [ ] 저장 완료 시 "기록이 저장되었습니다" 토스트 알림이 표시된다
- [ ] 임시저장된 기록은 '임시저장' 상태로 별도 표시되며 나중에 이어 작성할 수 있다
- [ ] 저장된 기록에는 작성자명, 역할, 작성 일시가 자동으로 기록된다

---

**US-022** — 기록 첨부파일 업로드
> 보호자로서, 병원 진단서·검사 결과지 등을 기록에 첨부하고 싶다. 그래야 기관에 전달할 때 기록과 원본 서류를 함께 공유할 수 있다.

**수락 기준:**
- [ ] 기록 작성 화면에서 파일 첨부 버튼으로 파일을 업로드할 수 있다
- [ ] 지원 형식: PDF, JPG, PNG, HEIC, DOCX (최대 20MB)
- [ ] 모바일에서 카메라 직접 촬영 또는 앨범 선택으로 첨부할 수 있다
- [ ] 업로드된 파일의 EXIF 메타데이터(GPS 등)가 자동으로 제거된다
- [ ] 파일 업로드 진행률이 표시된다
- [ ] 동일 기록에 최대 10개까지 첨부 가능하다
- [ ] 악성 파일 검사 후 업로드 거부 시 "허용되지 않는 파일 형식입니다" 메시지가 표시된다

---

**US-023** — 당사자 직접 기록 (쉬운 모드)
> 성인 당사자로서, 오늘 있었던 일이나 내 기분을 간단하게 직접 기록하고 싶다. 그래야 내 생각과 선호가 공식 기록에 남을 수 있다.

**수락 기준:**
- [ ] 당사자 화면에서 "오늘 기록하기" 버튼이 크게 표시된다 (최소 56px 높이)
- [ ] 이모지로 기분 선택 가능 (😊 😐 😢 😡 😴)
- [ ] 텍스트 입력과 함께 사진 첨부가 가능하다
- [ ] 미리 정의된 선택지(밥 먹기, 친구 만나기, 운동하기 등) 체크로도 기록 가능하다
- [ ] 저장 완료 시 "잘 저장됐어요! 👍" 피드백이 표시된다
- [ ] 기본 폰트 크기 18px 이상이 적용된다

---

### 6.4 타임라인 및 조회

**US-030** — 생애주기 타임라인 조회
> 보호자로서, 자녀의 모든 기록을 생애주기별 타임라인으로 한눈에 보고 싶다. 그래야 특정 시기의 상황을 빠르게 파악하고 기관에 설명할 수 있다.

**수락 기준:**
- [ ] 타임라인 화면에서 생애주기 구간별로 기록이 시간순으로 표시된다
- [ ] 카테고리 필터(전체/건강·의료/교육·발달/일상생활/서비스 연계/자기표현)로 필터링 가능하다
- [ ] 날짜 범위 지정 검색이 가능하다
- [ ] 키워드 검색으로 기록 내용을 검색할 수 있다
- [ ] 기록 카드에 카테고리 아이콘, 날짜, 작성자, 첨부파일 수, 공개범위가 표시된다
- [ ] 내 역할에서 열람 권한이 없는 기록은 타임라인에 표시되지 않는다

---

**US-031** — 기록 상세 조회 및 수정
> 사회복지사로서, 내가 작성한 기록을 나중에 다시 보고 필요 시 수정하고 싶다. 그래야 상황이 변한 경우 기록을 최신 상태로 유지할 수 있다.

**수락 기준:**
- [ ] 기록 상세 화면에서 모든 필드 값과 첨부파일 목록이 표시된다
- [ ] 내가 작성한 기록은 수정 버튼이 표시된다 (타인 작성 기록은 수정 불가)
- [ ] 기록 수정 시 이전 버전이 이력으로 저장된다
- [ ] 기록 삭제 시 "삭제 후 복구가 불가능합니다. 삭제하시겠습니까?" 확인 대화상자가 표시된다
- [ ] 삭제된 기록은 소프트 딜리트 처리되며 관리자는 30일 이내 복구 가능하다

---

### 6.5 기록자 권한 관리

**US-040** — 기록자 초대
> 보호자로서, 자녀의 특수교사를 플랫폼에 초대하여 IEP 기록 작성을 요청하고 싶다. 그래야 여러 기관의 전문가가 하나의 플랫폼에서 협력할 수 있다.

**수락 기준:**
- [ ] '기록자 관리' 화면에서 이메일로 기록자를 초대할 수 있다
- [ ] 초대 시 역할(사회복지사/특수교사/활동지원사 등)과 접근 가능 카테고리를 지정할 수 있다
- [ ] 접근 기간(종료일)을 선택적으로 지정할 수 있다
- [ ] 초대 이메일이 발송되며 초대 링크는 7일간 유효하다
- [ ] 초대 수락 전 상태는 "대기 중"으로 표시되며 취소 가능하다
- [ ] 초대 링크 만료 시 "만료된 초대입니다. 보호자에게 재요청하세요" 메시지가 표시된다
- [ ] 만 18세 미만 당사자의 경우 보호자만 기록자를 초대할 수 있다

---

**US-041** — 기록자 권한 수정 및 회수
> 성인 당사자로서, 내 기록에 접근하는 사람의 권한을 내가 직접 조정하거나 회수하고 싶다. 그래야 내 정보를 내가 통제할 수 있다.

**수락 기준:**
- [ ] 만 18세 이상 당사자는 기록자 목록을 조회하고 권한을 수정·회수할 수 있다
- [ ] 권한 회수 시 해당 기록자는 즉시 접근 불가 처리된다
- [ ] 권한 회수 이유를 선택적으로 입력할 수 있다
- [ ] 권한 회수 이벤트는 permission_logs에 영구 기록된다
- [ ] 회수된 기록자에게 "접근 권한이 변경되었습니다" 이메일 알림이 발송된다

---

**US-042** — 기록 공개범위 설정
> 보호자로서, 특정 기록은 특정 기록자에게만 보이도록 설정하고 싶다. 그래야 민감한 의료 기록을 모든 기록자가 볼 수 없게 제한할 수 있다.

**수락 기준:**
- [ ] 기록 작성 및 수정 시 공개범위를 선택할 수 있다: 나만보기 / 제한공개(기록자만) / 공유(초대된 모두)
- [ ] 기본값은 '제한공개'이다
- [ ] 공개범위 변경은 기록 작성자와 보호자(만 18세 미만 당사자 케이스)만 할 수 있다
- [ ] 공개범위가 '나만보기'인 기록은 타임라인에서 자물쇠 아이콘과 함께 다른 기록자에게 표시되지 않는다

---

### 6.6 권한 이양 플로우

**US-050** — 만 18세 권한 이양 알림
> 보호자로서, 자녀가 만 18세가 되기 전 권한 이양에 관한 안내를 미리 받고 싶다. 그래야 자녀와 충분히 상의하고 준비할 수 있다.

**수락 기준:**
- [ ] 만 18세 생일 D-7에 보호자와 당사자(앱 계정 있는 경우)에게 이양 안내 이메일이 발송된다
- [ ] 앱 내 알림 배너로 "홍길동님이 7일 후 만 18세가 됩니다"가 표시된다
- [ ] 이양 안내 화면에서 변경 사항(보호자 쓰기 권한 → 열람 전용 전환 등)이 명확히 설명된다
- [ ] 법정후견인 해당 여부를 확인하는 버튼이 제공된다

---

**US-051** — 권한 이양 프로세스 완료
> 성인 당사자로서, 만 18세가 되어 내 기록의 주 기록자가 되고 싶다. 그래야 나 스스로 기록을 관리하고 결정권을 행사할 수 있다.

**수락 기준:**
- [ ] 만 18세 생일 당일 시스템이 자동으로 TRANSITION_NOTIFIED 상태로 전환한다
- [ ] 당사자가 이양 시작 버튼 클릭 후 동의서 내용을 확인하고 서명할 수 있다
- [ ] 전자서명 후 PASS 본인인증으로 신원을 확인한다
- [ ] 인증 완료 시 보호자 쓰기 권한이 열람 전용으로 자동 전환된다
- [ ] 당사자에게 모든 권한이 부여되고 축하 화면이 표시된다
- [ ] 이양 완료 이벤트가 permission_logs에 영구 기록된다

---

**US-052** — 법정후견인 예외 처리
> 시스템 관리자로서, 법원의 성년후견·한정후견 결정을 플랫폼에 등록하여 당사자 권한 이양 예외 처리를 수행하고 싶다. 그래야 법적 상황을 플랫폼이 정확하게 반영할 수 있다.

**수락 기준:**
- [ ] 관리자 화면에서 당사자별 후견 여부와 후견 유형(성년/한정)을 등록할 수 있다
- [ ] 법정후견 등록 시 해당 당사자는 GUARDIAN_OVERRIDE 상태로 전환된다
- [ ] 한정후견의 경우 후견 범위 내에서만 후견인이 기록 작성 권한을 가진다
- [ ] 후견 변경 이력이 permission_logs에 기록된다

---

### 6.7 PDF 출력 및 공유

**US-060** — 기록 요약 PDF 출력
> 보호자로서, 기관 전환 시 필요한 핵심 기록만 선택하여 PDF로 출력하고 싶다. 그래야 기관 담당자에게 깔끔한 형태로 정보를 전달할 수 있다.

**수락 기준:**
- [ ] 출력할 카테고리, 생애주기 구간, 날짜 범위를 선택할 수 있다
- [ ] 수신 기관명과 제공 목적을 입력하여 PDF 상단에 표시할 수 있다
- [ ] 민감 정보(장애인등록번호 마지막 4자리 등) 마스킹 옵션을 제공한다
- [ ] PDF 생성은 백그라운드로 처리되며 완료 시 알림이 표시된다
- [ ] 생성된 PDF는 15분 내 다운로드 가능한 링크로 제공된다
- [ ] PDF 출력 이벤트가 접근 로그에 기록된다

| AUTH-P0-6 | RBAC 권한 가드 | 역할 외 권한 접근 시 403 반환. 모든 API 엔드포인트 적용 |

## 15. 타임라인

### 15.1 마일스톤 개요

```
M1 (1~2월)    M2 (2~3월)    M3 (3~4월)    M4 (4~5월)    M5 (5~6월)    M6 (6월)
────────────────────────────────────────────────────────────────────────────────
기반 구축      템플릿 엔진    권한 이양       UI/UX 완성    파일럿 PoC    정식 오픈
              기록 CRUD     알림 시스템     베타 배포
```

### 15.2 단계별 상세 일정

#### M1 (1~2개월차) — 기반 구축
- [ ] 기술 스택 확정 및 개발 환경 구성 (Docker, CI/CD, GitLab)
- [ ] DB 스키마 설계 확정 및 마이그레이션 스크립트
- [ ] 인증 모듈: 회원가입, 로그인, JWT, 이메일 인증
- [ ] RBAC 엔진 및 권한 가드 구현
- [ ] 생애주기 판별 서비스 구현
- [ ] 전문인력 기관코드 인증 플로우
- [ ] 법무 자문 (권한 이양 관련) 완료

**체크포인트:** 역할별 계정 생성 후 대시보드 진입 성공

#### M2 (2~3개월차) — 템플릿 엔진 & 기록 CRUD
- [ ] 25개 이상 기록 템플릿 마스터 데이터 설계·입력
- [ ] 동적 폼 렌더링 엔진 (10가지 필드 타입)
- [ ] 기록 CRUD API 전체 (생성/조회/수정/삭제)
- [ ] 파일 업로드/다운로드 (MinIO 연동, EXIF 제거)
- [ ] 역할별 권한 필터링 미들웨어
- [ ] 기록자 초대·수락·회수 플로우
- [ ] 파일럿 기관 MOU 체결 완료

**체크포인트:** 역할별 기록 작성 → 저장 → 조회 E2E 성공

#### M3 (3~4개월차) — 권한 이양 시스템
- [ ] 만 18세 자동 전환 스케줄러 (Cron)
- [ ] D-7 사전 알림 이메일·앱 내 알림
- [ ] 전자 동의서 서명 UI + 토큰 검증
- [ ] PASS 본인인증 연동 (또는 이메일 OTP 대안)
- [ ] 후견인 예외 처리 로직 (관리자 수동 등록)
- [ ] 권한 이양 상태 대시보드 (보호자·당사자 화면)
- [ ] 이양 완료 알림 및 보호자 권한 자동 조정

**체크포인트:** 만 18세 시뮬레이션 → 이양 플로우 전체 E2E 통과

#### M4 (4~5개월차) — UI/UX 완성 & 베타 배포
- [ ] 역할별 대시보드 전체 완성
- [ ] 생애주기 타임라인 뷰 구현
- [ ] 기록 검색·필터 기능
- [ ] PDF 출력 기능 (Puppeteer 연동)
- [ ] 당사자 쉬운 모드 UI 구현
- [ ] 접근성 점검 및 개선 (WCAG 2.1 AA)
- [ ] 베타 버전 스테이징 환경 배포

**체크포인트:** 스테이징 환경에서 주요 시나리오 전체 QA 통과

#### M5 (5~6개월차) — 파일럿 PoC 운영
- [ ] 파일럿 기관 3곳 온보딩 (관리자 교육, 계정 셋업)
- [ ] 실 사용자 2주 이상 운영
- [ ] 사용자 피드백 수집 및 주요 버그 수정
- [ ] 외부 보안 감사 (취약점 점검) 의뢰 및 완료
- [ ] 개인정보 영향평가 (PIA) 착수

**체크포인트:** 파일럿 기관 3곳 PoC 리포트 완성

#### M6 (6개월차) — 정식 오픈
- [ ] 보안 감사 취약점 전체 조치
- [ ] PIA 완료 및 검토 결과 반영
- [ ] k6 기반 성능 부하 테스트 (동시 500명)
- [ ] 운영 모니터링 대시보드 (Prometheus + Grafana) 완비
- [ ] 공식 서비스 도메인 DNS 설정 및 SSL 인증서
- [ ] 정식 서비스 론칭
- [ ] B2G 수주 제안서 작성 (PoC 결과 기반)

**체크포인트:** 정식 서비스 오픈 & 월간 가동률 99.5% 달성
